## 平台适配目标

Task233 Runtime 不依赖 worker thread、反射生成、动态代码或 unsafe，调度基于 Unity PlayerLoop，目标覆盖 Unity 2022+ 的 Mono / IL2CPP / WebGL 单线程平台。当前 package 是 Unity-first 运行时；取消句柄和 awaitable 形状保持 .NET Standard 友好，后续可拆出传统 C# 通用 scheduler。

## 运行方式

先下载 UniTask 源码包到本地 `.perf/UniTask`：

```powershell
powershell -ExecutionPolicy Bypass -File Tools/PrepareUniTaskPackage.ps1 -Version 2.5.11
```

再运行 Unity 2022.3.51f1：

```powershell
"C:\Program Files\Unity\Hub\Editor\2022.3.51f1\Editor\Unity.exe" -batchmode -nographics -projectPath TestProject -runTests -testPlatform EditMode -testResults artifacts\editmode-results.xml
```

GitHub Actions 会下载 UniTask `2.5.11` 源码包、创建隔离测试工程，并运行同一套测试。仓库没有配置 Unity license secret 时，CI 会通过配置校验并跳过真实 Unity Editor 测试。
