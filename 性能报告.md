## 对比表

Unity Performance Testing 的 `Time` 样本是每个 measurement 的耗时。本表按测试中的 `IterationsPerMeasurement` 折算为 `ns/op`。

| 场景 | Task233 API | UniTask API | Task233 ns/op | UniTask ns/op | 倍率 | Task233 GC median | UniTask GC median |
| --- | --- | --- | ---: | ---: | ---: | ---: | ---: |
| Continuation 入队 | `T233.Post(cachedAction)` | 无直接等价项 | 2100.00 | N/A | N/A | 0.0000 | N/A |
| Yield factory | `T233.Yield()` | `UniTask.Yield()` | 8.87 | 1.48 | UniTask 6.0x | 0.0003 | 0.0001 |
| 等待 1 帧 factory | `T233.DelayFrames(1)` | `UniTask.DelayFrame(1)` | 13.10 | 157.53 | Task233 12.0x | 0.0003 | 1.0003 |
| 等待秒 factory | `T233.DelaySeconds(0.001d)` | `UniTask.Delay(TimeSpan.FromSeconds(...))` | 13.50 | 197.04 | Task233 14.6x | 0.0003 | 1.0003 |
| 等待毫秒 factory | `T233.DelayMilliseconds(1)` | `UniTask.Delay(1)` | 12.93 | 206.39 | Task233 16.0x | 0.0003 | 1.0003 |

说明：`GC median` 是 Unity Performance Testing `Measure.Method(...).GC()` 输出的 `Time.GC()` 样本中位数。Unity 在 XML 中将该单位标为 `Undefineds`，因此这里按原始数值记录，用于同环境横向比较。

## 运行方式

1. 在 `TestProject/Packages/manifest.json` 中加入 UniTask 2.5.11。
2. `Task233.UniTaskPerformanceTests.asmdef` 会通过 Version Defines 自动开启 `TASK233_HAS_UNITASK`。
3. 配置 GitHub secrets：`UNITY_LICENSE` 或 `UNITY_SERIAL`。
4. 运行 GitHub Actions：`Unity Performance Tests`。

也可以本地运行：

```powershell
"C:\Program Files\Unity\Hub\Editor\2022.3.51f1\Editor\Unity.exe" -batchmode -nographics -projectPath TestProject -runTests -testPlatform EditMode -testResults artifacts\editmode-results.xml
```
