Task233 Unity

Unity 2022+ async scheduling primitives built for zero-GC hot paths, explicit time-unit delays, lightweight cancellation, WebGL-friendly single-thread execution, and measurable performance work.

Install

https://github.com/neko233-com/Task233-unity.git

For offline projects, create Assets/neko233/Task233 and copy only Runtime and Editor into it.

Why

Low overhead

Struct awaitables and pooled scheduler nodes avoid coroutine allocations for common frame and time waits after prewarm.

Unity native

PlayerLoop integration runs on Unity's main thread without requiring a scene object.

Benchmarkable

Performance tests are included, with an optional UniTask comparison assembly for direct speed and GC checks in under a minute.

Platform fit

No worker thread, unsafe, reflection emit, or dynamic code generation requirement for Unity 2022+ runtime platforms.

Measured Performance

Unity 2022.3.51f1 EditMode, WindowsEditor, no warmup, one measurement, total time converted to throughput. Full data: 性能报告.md.

Yield

1.249 ns/op for T233.Yield(), slightly faster than UniTask in the same short run.

Frame delay

18.3x faster than UniTask.DelayFrame(1) factory creation.

Time delay

19.8x to 21.3x faster for seconds and milliseconds delay factory creation.

Cancellation

3.8x faster than CancellationTokenSource create/cancel/dispose.

Quick Start

using Task233;
using UnityEngine;

public sealed class Example : MonoBehaviour
{
    private async void Start()
    {
        await T233.Yield();
        await T233.DelayFrames(3);
        await T233.DelaySeconds(0.25d);
        await T233.DelayMilliseconds(16);
        Debug.Log("continued on Unity main thread");
    }
}

Cancellation

var cancel = T233.CreateCancelSource();

try
{
    await T233.DelaySeconds(3, cancellation: cancel);
}
finally
{
    cancel.Dispose();
}
